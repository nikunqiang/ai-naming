# 中文取名智能体设计文档

> 创建日期：2026-04-12

## 一、项目概述

### 1.1 项目背景

为新生儿父母提供专业的中文取名服务，结合传统文化（诗经、楚辞、五行八字）与现代审美，帮助家长为宝宝取一个寓意美好的名字。

### 1.2 目标用户

- 新生儿父母
- 注重文化内涵的家庭
- 希望名字有传统根基又符合现代审美

### 1.3 核心价值

- **文化底蕴**：取材诗经、楚辞等经典
- **传统智慧**：五行八字、三才五格分析
- **现代审美**：音韵优美、易读易写
- **灵活配置**：用户可选择传统/文学/现代/混合模式

### 2.2 入口设计

| 入口 | 说明 |
|------|------|
| **取名** | 为宝宝取新名字 |
| **测名** | 分析已有名字 |

### 2.3 信息收集字段

| 字段 | 必填 | 说明 |
|------|------|------|
| 父亲姓氏 | ✅ | 单姓或复姓 |
| 母亲姓氏 | ❌ | 可选，用于组合取名 |
| 宝宝性别 | ✅ | 男/女/未定 |
| 出生时间 | ❌ | 用于八字五行计算 |
| 期望寓意 | ❌ | 如"聪明、健康、有文采" |
| 避免用字 | ❌ | 长辈名字、忌讳字等 |
| 字数偏好 | ❌ | 单字名/双字名/不限 |

### 2.4 取名模式

| 模式 | 说明 | 分析重点 |
|------|------|----------|
| 传统 | 八字五行 + 三才五格 + 生肖宜忌 | 注重传统命名学 |
| 文学 | 诗经楚辞取意 + 音韵美感 | 注重文化内涵 |
| 现代 | 寓意优先 + 避免生僻字 + 易读易写 | 注重实用性 |
| 混合 | 以上全部考虑 | 默认推荐 |

### 2.5 名字分析项

| 分析项 | 说明 | 数据来源 |
|--------|------|----------|
| 字义解析 | 每个字的基本含义、引申义 | 字库 + LLM |
| 五行分析 | 每个字的五行属性、整体五行组合 | 字库 |
| 笔画分析 | 简体笔画、康熙笔画、三才五格数理 | 字库计算 |
| 音律分析 | 声调搭配、韵母组合、读音流畅度 | 字库 + 规则 |
| 字形分析 | 字形结构、繁简体、生僻程度 | 字库 |
| 诗词典故 | 名字出自哪句诗词、文化内涵 | RAG 检索诗经楚辞 |
| 寓意解读 | 整体寓意、美好祝愿 | LLM |
| 谐音检查 | 是否有不雅谐音 | LLM |

用于八字五行计算的开源库，功能包括：

- 八字排盘（年柱、月柱、日柱、时柱）

// 公历转农历

console.log(bazi.getYear())  // 年柱
console.log(bazi.getMonth()) // 月柱

#### 三才五格算法

参考 fate 项目的 Go 实现，核心算法：

```typescript
// 五格计算
interface WuGe {
  tianGe: number  // 天格
  renGe: number   // 人格
  diGe: number    // 地格
  waiGe: number   // 外格
  zongGe: number  // 总格
}

// 天格：复姓为姓的笔画相加，单姓为姓的笔画加一
function calcTianGe(l1: number, l2: number): number {
  return l2 === 0 ? l1 + 1 : l1 + l2
}

// 人格：复姓为姓的第二字+名的第一字，单姓为姓+名的第一字
function calcRenGe(l1: number, l2: number, f1: number): number {
  return l2 !== 0 ? l2 + f1 : l1 + f1
}

// 地格：复名为名相加，单名为名+1
function calcDiGe(f1: number, f2: number): number {
  return f2 === 0 ? f1 + 1 : f1 + f2
}

// 外格：根据姓氏和名字情况计算
function calcWaiGe(l1: number, l2: number, f2: number): number {
  if (l2 === 0 && f2 === 0) return 2      // 单姓单名
  if (l2 === 0 && f2 !== 0) return 1 + f2 // 单姓复名
  if (l2 !== 0 && f2 === 0) return l1 + 1 // 复姓单名
  return l1 + f2                           // 复姓复名
}

// 总格：所有笔画相加
function calcZongGe(l1: number, l2: number, f1: number, f2: number): number {
  return l1 + (l2 || 0) + f1 + (f2 || 0)
}

// 三才属性：根据五格数推算五行
// 1-2木，3-4火，5-6土，7-8金，9-10水
function getSanCaiAttr(num: number): string {
  const sanCai = '水木木火火土土金金水'
  return sanCai[num % 10]
}
```

### 4.6 数据处理说明

**五行解码**：

cnchar 的五行数据使用编码方式存储，需要在代码中解码：

```typescript
// 五行编码映射
const WUXING_MAP: Record<string, string> = {
  'a': '金', 'b': '木', 'c': '水', 'd': '火', 'e': '土', 'f': '-'
}

// 解码汉字五行
function getWuxing(char: string, charInfo: Record<string, string>): string {
  const info = charInfo[char]
  if (!info) return '-'
  const wuxingCode = info[1] // 第二个字符为五行主属性
  return WUXING_MAP[wuxingCode] || '-'
}

// 示例
getWuxing('明', charInfo) // 返回 '火'（编码 'd'）
```

**笔画查询**：

```typescript
// 按笔画数查询汉字
function getCharsByStroke(strokeCount: number, strokeData: Record<string, string>): string[] {
  const chars = strokeData[strokeCount.toString()]
  return chars ? chars.split('') : []
}

// 示例
getCharsByStroke(8, strokeCount) // 返回 8 画的汉字列表
```

## 五、前端设计

### 5.1 设计理念

**设计风格**：现代中式简约

以中国传统水墨美学为灵感，融合现代极简设计语言。留白、呼吸感、文化底蕴是核心关键词。界面克制而不冷淡，优雅而不张扬。

**设计原则**：
- **留白即美**：大量负空间，让内容呼吸
- **克制动效**：只在关键节点使用微动效，不喧宾夺主
- **文化点缀**：朱砂红作为点睛之笔，水墨灰营造氛围
- **功能优先**：每个元素都有明确目的，无装饰性冗余

#### 5.5.2 装饰动效

## 七、经典名字赏析与取名策略

### 7.1 经典名字案例

#### 7.1.1 出自诗经楚辞

| 名字 | 出处 | 寓意 | 特点 |
|------|------|------|------|
| **扶苏** | 《诗经·郑风·山有扶苏》"山有扶苏，隰有荷华" | 良木之名，象征茂盛挺拔、生机勃勃 | 双字名，字义搭配独特，不落俗套 |
| **静姝** | 《诗经·邶风·静女》"静女其姝，俟我于城隅" | 娴静美好的女子 | 专用于女孩，典雅端庄 |
| **望舒** | 《楚辞·离骚》"前望舒使先驱兮" | 神话中为月亮驾车的神，代指月亮 | 神话人物入名，浪漫飘逸 |
| **怀瑾** | 《楚辞·九章·怀沙》"怀瑾握瑜兮" | 怀揣美玉，比喻怀抱美德 | 以玉喻德，君子之风 |
| **婉清** | 《诗经·郑风·野有蔓草》"有美一人，清扬婉兮" | 温婉美好，清雅脱俗 | 女孩名常用组合，典雅大方 |
| **琼瑶** | 《诗经·卫风·木瓜》"投我以木瓜，报之以琼瑶" | 美玉，喻珍贵、美好 | 双玉组合，晶莹剔透 |

#### 7.1.2 出自诗词

| 名字 | 出处 | 寓意 | 特点 |
|------|------|------|------|
| **疏影** | 林逋《山园小梅》"疏影横斜水清浅，暗香浮动月黄昏" | 梅花疏淡清雅的姿态 | 以景入名，画面感强，意境深远 |
| **徽因** | 《诗经·大雅·思齐》"大姒嗣徽音，则百斯男" | 美好的声誉 | 三字名音韵协调 |

#### 7.1.3 名人名字

| 名字 | 分析 | 借鉴要点 |
|------|------|----------|
| **苏轼**（字子瞻） | 轼：车前横木；子瞻：远望。名与字呼应——凭轼而望 | 名与字可形成呼应关系 |
| **李白**（字太白） | 太白金星，传说其母梦见太白金星入怀而生 | 名字可与传说、意象关联 |
| **莫言** | 笔名，"少说话，多做事"，与其文学风格呼应 | 笔名可取反义或否定词，形成张力 |

### 7.2 好名字的共同特征

#### 7.2.1 音韵优美

| 原则 | 说明 | 示例 |
|------|------|------|
| 声调搭配 | 避免全同声调，平仄相间 | 扶苏（阳平+阴平）、静姝（去声+阴平） |
| 韵母搭配 | 避免韵母相同或相近 | 疏影、婉清 |
| 开闭口音 | 开闭搭配，抑扬顿挫 | 望舒、怀瑾 |

#### 7.2.2 寓意美好

| 类型 | 说明 | 示例 |
|------|------|------|
| 直接寓意 | 字义明确美好 | 静姝、婉清 |
| 借物喻志 | 以物喻人格理想 | 扶苏（良木）、怀瑾（美玉） |
| 意境入名 | 画面感、氛围感 | 疏影、望舒 |

#### 7.2.3 文化底蕴

| 类型 | 说明 | 示例 |
|------|------|------|
| 诗经出典 | 典雅端庄 | 静姝、婉清、琼瑶 |
| 楚辞出典 | 浪漫飘逸 | 望舒、怀瑾 |
| 诗词出典 | 意境深远 | 疏影 |

#### 7.2.4 不落俗套

| 原则 | 说明 | 示例 |
|------|------|------|
| 用字独特 | 不常见但易读 | 扶苏、疏影 |
| 组合新颖 | 常见字新组合 | 怀瑾、望舒 |
| 避免重名 | 不易与他人重名 | 莫言、扶苏 |

### 7.3 取名生成策略

#### 策略一：典籍取意

```
1. 从诗经/楚辞中检索符合期望寓意的诗句
2. 从诗句中提取关键字
3. 组合成名，确保音韵和谐
4. 验证无谐音问题
```

**示例**：
- 期望寓意：温婉美好
- 检索诗句：《诗经·郑风·野有蔓草》"有美一人，清扬婉兮"
- 提取关键字：婉、清
- 组合成名：婉清
- 验证：音韵和谐，无谐音问题 ✓

#### 策略二：字库筛选

```
1. 根据五行/笔画筛选候选字
2. 根据性别过滤（男孩/女孩/通用）
3. 根据寓意过滤（期望寓意）
4. 组合成名，验证音韵和谐
```

**示例**：
- 五行需求：缺水
- 性别：女孩
- 筛选结果：清、洁、涵、润、淑...
- 组合成名：清涵、淑润...

#### 策略三：混合策略（推荐）

```
1. 典籍取意确定名字风格和文化底蕴
2. 字库筛选确保五行笔画符合
3. 组合优化，确保音韵和谐
4. 最终验证，排除谐音歧义
```

### 7.4 避坑指南

| 风险 | 说明 | 示例 |
|------|------|------|
| 谐音问题 | 检查是否有不雅谐音 | 杜子腾→肚子疼、范统→饭桶 |
| 生僻字 | 避免过于生僻的字 | 彧、翀等，影响日常使用 |
| 重名率高 | 避免过于常见的名字 | 子轩、子涵等高频名 |
| 歧义联想 | 避免有歧义的字 | 菊、花等可能有歧义 |

详见 `data/classic_names_analysis.md`，包含：
