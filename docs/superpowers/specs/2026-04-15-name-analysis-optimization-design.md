# 名字分析逻辑全量升级设计

> 目标：全面提升名字分析质量，使分析结果更符合主流审美，以传统命理优先

## 一、评分体系重构

### 新评分权重（100分制）

| 维度 | 分值 | 说明 |
|------|------|------|
| 五行配置 | 25 | 八字五行缺失补益 + 名字五行相生相克关系 |
| 三才五格 | 20 | 五格数理吉凶 + 三才配置吉凶 |
| 音律听感 | 20 | 声调搭配 + 韵母搭配 + 开闭口音 + 谐音检查 |
| 字义内涵 | 20 | 字义吉凶 + 文化出处 + 寓意深度 |
| 字形结构 | 10 | 笔画平衡 + 书写便利 + 视觉美感 + 生僻度 |
| 时代适用 | 5 | 重名率 + 现代接受度 |

### 评分锚定规则

- 90+ 分：各维度都优秀，极少数名字能达到
- 80-89 分：多数维度优秀，个别有瑕疵
- 70-79 分：普通好名字，有一定优点
- 60-69 分：有较明显问题
- <60 分：有严重问题

### 各维度扣分/加分规则

**五行配置（25分）**：
- 名字五行与八字喜用神匹配 → 每个匹配 +8分
- 名字五行包含忌神 → 每个冲突 -8分
- 名字五行补了八字缺失 → 每个补益 +5分
- 名字五行与八字五行相克 → 每个冲突 -5分
- 无八字信息时，按字面五行平衡度评分，上限15分

**三才五格（20分）**：
- 三才大吉 → 20分
- 三才半吉 → 14分
- 三才凶 → 6分
- 人格（核心格）吉 → +3分；凶 → -5分
- 总格吉 → +2分；凶 → -3分

**音律听感（20分）**：
- 声调全同 → -8分
- 有不良谐音 → -10分
- 韵母冲突 → -5分
- 相邻字同声调 → -3分
- 末字为去声（收音有力）→ +2分

**字义内涵（20分）**：由 LLM 评估，但有锚定规则

**字形结构（10分）**：
- 笔画标准差 > 5 → -3分
- 单字笔画 > 20 → -2分
- 同结构类型连续出现 → -1分
- 生僻字 → -3分

**时代适用（5分）**：
- 重名率极高 → -3分
- 重名率高 → -1分

## 二、音韵分析增强

### 2.1 声调分析细化

| 声调搭配 | 评价 | 分值调整 |
|----------|------|----------|
| 平仄相间（如 阴平+上声+去声） | 抑扬顿挫，最佳 | 0 |
| 两平一仄或两仄一平 | 有变化，良好 | -2 |
| 全平或全仄 | 平淡/生硬 | -5 |
| 相邻两字同声调 | 缺乏变化 | -3 |
| 末字为去声 | 收音有力 | +2 |

### 2.2 韵母搭配分析（新增）

- 相邻字韵母相同 → -3分
- 相邻字韵母相近（an/ang, en/eng, in/ing）→ -1分

### 2.3 开闭口音分析（新增）

- 开口音：a, o, e, ai, ei, ao, ou, an, en, ang, eng
- 闭口音：i, u, ü, in, un, ing
- 全开口或全闭口 → -2分
- 开闭搭配 → 不扣分

### 2.4 谐音检查扩展

- 从8条扩展至50+条普通话谐音规则
- 增加"同音不同调"检查
- 增加"近音"检查（声母韵母相同）
- 谐音词库独立为 `data/harmony_warnings.json`

### 2.5 输出结构

```typescript
interface PhoneticResult {
  pinyins: string[]
  tones: number[]
  toneNames: string[]
  toneAnalysis: string      // 声调评价
  rhymeAnalysis: string     // 韵母评价（新增）
  opennessAnalysis: string  // 开闭口音评价（新增）
  analysis: string          // 综合评价
  score: number             // 音律得分（满分20）
}
```

## 三、五行与三才五格增强

### 3.1 八字五行补益分析（新增）

新增 `analyzeWuxingBenefit(baZi, nameWuxing)` 函数：

1. **确定八字喜用神**：
   - 日主强 → 喜克泄耗（官杀、食伤、财星对应的五行）
   - 日主弱 → 喜生扶（印星、比劫对应的五行）
   - 简化实现：按五行缺失 + 日主五行强弱判断

2. **名字五行补益评分**：按评分规则执行

3. **五行相生相克关系**：
   - 相邻字五行相生（如木→火）→ +3分
   - 相邻字五行相克（如木→金）→ -3分
   - 同五行 → 中性

### 3.2 三才五格数理吉凶细化

1. 新增五格数理吉凶表（1-81数的吉凶判定）
2. 输出增加每个格的吉凶判定
3. 三才五格量化评分

### 3.3 输出结构

```typescript
interface WuxingBenefitResult {
  xiYong: string[]      // 喜用神五行
  jiShen: string[]      // 忌神五行
  nameBenefit: string   // 名字对八字的补益说明
  score: number         // 补益得分（满分25）
}

interface WuGeDetail {
  tianGe: { num: number, wuxing: string, fortune: string }
  renGe: { num: number, wuxing: string, fortune: string }
  diGe: { num: number, wuxing: string, fortune: string }
  waiGe: { num: number, wuxing: string, fortune: string }
  zongGe: { num: number, wuxing: string, fortune: string }
}
```

## 四、RAG 升级 + 数据源扩展

### 4.1 嵌入模型替换

- 从字频向量替换为 `bge-m3` 本地嵌入模型（通过 Ollama 部署）
- `ollama pull bge-m3`，通过 Ollama `/api/embeddings` 接口调用
- 启动时预计算所有经典条目的嵌入向量，缓存到本地 JSON 文件
- 备选：远程嵌入 API（如讯飞嵌入服务）

### 4.2 经典文献扩展

新增数据源（来自 `chinese-poetry` 开源项目）：
- 唐诗三百首（约300首）
- 宋词三百首（约300首）
- 论语名句（约200条，精选适合取名的句子）

### 4.3 检索策略优化

- 语义搜索 + 关键词搜索混合：语义 top 20 + 关键词 top 5，去重取 top 10
- 按主题分类检索：文学模式优先诗词，传统模式优先典籍
- 检索结果增加相关度评分展示

### 4.4 重名率分析（新增）

- 数据源：`names_corpus_gender.txt`（120万条）
- 启动时加载词频统计，构建 `{名字: 出现次数}` 的 Map
- `analyzeNamePopularity(name)` 返回重名等级和同音名数量

## 五、AI 提示词优化

### 5.1 评分锚定

将所有预计算的量化评分直接传入提示词，LLM 角色从"打分者"变为"解释者+微调者"：

- LLM 在量化评分基础上微调 ±2 分，必须说明理由
- 综合评分 = 各维度评分之和 + 微调

### 5.2 结构化提示词模板

```
## 预计算数据
- 音律得分：{phoneticScore}/20
- 五行补益得分：{wuxingScore}/25
- 三才五格得分：{sancaiScore}/20
- 字义得分：{meaningScore}/20（待评估）
- 字形得分：{glyphScore}/10
- 时代适用得分：{eraScore}/5

## 输出格式
### 综合评分：XX/100
### 各维度评分（表格）
### 详细分析
### 专家点评
### 改进建议
```

### 5.3 系统提示词增强

- 评分锚定规则写入系统提示词
- 禁止"和稀泥"：差就是差，不用"尚可""还行"
- 五行/三才五格解读必须引用具体数据

## 六、字形结构分析增强（新增模块）

### 6.1 笔画平衡度

- 标准差 ≤ 2 → 不扣分
- 标准差 3-5 → -1分
- 标准差 > 5 → -3分

### 6.2 书写便利度

- 单字笔画 > 20 → -2分
- 单字笔画 > 15 → -1分
- 名字总笔画 > 30 → -1分

### 6.3 视觉结构分析

- 新增字结构映射表（左右、上下、包围、独体等）
- 同结构类型连续出现 → -1分
- 结构多样 → 不扣分

### 6.4 生僻度检测

- 在取名常用字中 → 不扣分
- 在 GB2312 中但不在常用字中 → -1分
- 不在 GB2312 中 → -3分

## 七、整体数据流

```
用户输入姓名 + 出生时间
        │
        ▼
  ① 字符查询 ──→ getCharacterInfo() × 每个字
        │
        ▼
  ② 八字五行 ──→ calculateBaZi() → analyzeWuxingBenefit()
        │              计算喜用神、名字补益评分
        ▼
  ③ 三才五格 ──→ analyzeSanCaiWuGe()
        │              五格各数吉凶判定 + 量化评分
        ▼
  ④ 音律分析 ──→ analyzePhonetic()
        │              声调 + 韵母 + 开闭口音 + 量化评分
        ▼
  ⑤ 谐音检查 ──→ checkHarmony()
        │              50+条规则 + 近音检查
        ▼
  ⑥ 字形分析 ──→ analyzeGlyph()
        │              笔画平衡 + 书写便利 + 视觉结构 + 生僻度
        ▼
  ⑦ 重名率   ──→ analyzeNamePopularity()
        │              基于120万名字词频数据
        ▼
  ⑧ 经典检索 ──→ searchClassics()
        │              bge-m3嵌入 + 语义+关键词混合检索
        ▼
  ⑨ AI分析   ──→ 将所有量化评分传入提示词
                   LLM 解释评分 + 微调 ±2 + 生成报告
```

前端进度步骤从5步扩展为8步。
